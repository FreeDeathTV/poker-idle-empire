V901 – Global PWA Install Button
Overview
V901 introduces a global “Install App” button to enable a proper PWA installation flow.
The button appears in the header, positioned on the left side, directly under the “?” help button.

This version does not modify gameplay, economy, or layout structure beyond adding the install component.

Goals
Add a global PWA install button visible on all pages

Trigger the browser’s native install prompt

Remove the need for manual “Add to Home Screen”

Maintain zero drift: only touch header + install component

Scope
1. InstallPrompt Component
Create a small Svelte component that:

Listens for beforeinstallprompt

Stores the event

Shows an install button

Calls .prompt() when clicked

Hides itself after installation

No external libraries.

2. Header Integration
Import the InstallPrompt component into the global header

Position it left side, directly under the “?” help button

Use minimal styling consistent with the existing header UI

Ensure it does not overlap or break mobile layout

3. Manifest Requirements
Confirm the manifest includes:

json
"display": "standalone",
"start_url": "/"
No other manifest changes required.

Zero‑Drift Rules
Do NOT modify game logic

Do NOT modify stores

Do NOT modify routes

Do NOT modify Roulette

Do NOT modify economy

Do NOT modify layout structure beyond placing the button

Do NOT introduce new dependencies

Only add the install component and place it in the header.

Deliverables
InstallPrompt.svelte

Header updated with install button

Working PWA install flow

No layout regressions

No gameplay changes
